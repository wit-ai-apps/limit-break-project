# 映像視聴判定・毎朝自動再計画

## 目的

以前に理解した単元へ同じ映像時間を使い続けず、確認テストで現在の定着を確かめてから映像の扱いを決めます。同時に、提出が遅れても8月31日の期限から逆算し、毎朝計画を作り直します。

## 映像視聴判定

| 条件 | 判定 | 学習行動 |
|---|---|---|
| 信頼できる提出がない、結果が15日以上前、回答5問未満、AI信頼度65%未満 | 先に確認テスト | 10〜20分の診断後に映像を決定 |
| 正答率85%以上 | 映像を省略 | 15分の定着確認後、次単元へ |
| 正答率70〜84% | 要点映像のみ | 苦手箇所の映像と短い類題 |
| 正答率50〜69% | 通常視聴 | 映像授業、確認問題、間違い分類 |
| 正答率50%未満 | 基礎から再構築 | 前提単元、映像授業、基本問題 |

問題文や解答本文は計画データへ保存せず、教材名、講、PART、結果、判定理由だけを扱います。

## 毎朝8:55の再構築

Firebase Scheduled Functionsを `55 8 * * *`、タイムゾーン `Asia/Tokyo` で実行します。

1. 生徒に紐づく提出記録を取得
2. 教材・講・PART単位で最新の信頼できる結果を選択
3. 8月31日までの残日数と、週1日の調整日を除いた実学習日を計算
4. 完了単元を除いて、1日あたりに必要な単元数を再計算
5. 各教材を「確認先行・映像省略・要点のみ・通常視聴・基礎へ戻る」に分類
6. `students/{studentId}/adaptive_plans/{date}` と `adaptive_state/current` へ保存

直近2日間に提出がない場合は `no_recent_submission` とし、未提出を完了扱いせず診断テストを優先します。必要時間が1日420分を超える場合は `compressed` とし、保護者・講師にも圧縮状態を表示します。

## 更新タイミング

- 毎朝8:55 JST
- 提出画像または確認テスト記録が追加・更新された直後

## 現在の対象教材

- ベーシックレベル数学Ⅰ
- ベーシックレベル数学A
- ベーシックレベル英語
- スタンダードレベル英語 文法・読解

理科その他は、講・PART単位の総数を確定してから同じアルゴリズムへ追加します。
